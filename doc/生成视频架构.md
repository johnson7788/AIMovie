 Idea2Video 生成长视频逻辑的完整分析，结合了代码和缓存数据。

  ---
  整体架构

  Idea2Video 是一个文本到长视频的生成管线，由两层 Pipeline 组成：

  Idea2VideoPipeline (顶层编排)
      └── Script2VideoPipeline (逐场景生成)

  ---
  第一层：Idea2VideoPipeline — 7步编排

  入口文件 main_idea2video.py:25，接收 3 个输入：

  ┌──────────────────┬────────────────────────────────────────┐
  │       输入       │                  含义                  │
  ├──────────────────┼────────────────────────────────────────┤
  │ idea             │ 故事创意（自由文本）                   │
  ├──────────────────┼────────────────────────────────────────┤
  │ user_requirement │ 约束条件（如场景数、镜头数限制）       │
  ├──────────────────┼────────────────────────────────────────┤
  │ style            │ 视觉风格（如"storybook illustration"） │
  └──────────────────┴────────────────────────────────────────┘

  步骤 1：发展故事 (develop_story)

  Screenwriter.develop_story() 将 idea 扩展为完整故事，输出 story.txt。

  本例中输出了一个标题为 "橙纹小蝶的第一趟飞翔" 的故事，约 60 秒，4 段时间线（0s-12s / 12s-28s / 28s-45s / 45s-60s），包含 3 个角色。

  步骤 2：提取角色 (extract_characters)

  CharacterExtractor 从 story 中提取角色，输出 characters.json。

  本例提取了 3 个角色（characters.json:2-22）：
  - idx=0: Little Orange Stripe (蝴蝶) — static_features + dynamic_features（花粉、桂花屑）
  - idx=1: Tricolor napping kitten — static_features + dynamic_features
  - idx=2: Silver-haired grandma over 70 — static_features + dynamic_features

  步骤 3：生成角色肖像 (generate_character_portraits)

  为每个角色生成 3 个视角的角色参考图（正/侧/背），存到 character_portraits/ 目录：

  character_portraits/
      0_Little Orange Stripe.../    front.png (4MB) / side.png (3.8MB) / back.png (4.1MB)
      1_Tricolor napping kitten/   front.png (4.8MB) / side.png (3.3MB) / back.png (5.3MB)
      2_Silver-haired grandma.../ front.png (4MB)   / side.png (3.5MB) / back.png (3.6MB)

  生成顺序是：先生成正面 → 以正面为参考生成侧面 → 以正面为参考生成背面。角色肖像注册表写入 character_portraits_registry.json。

  步骤 4：基于故事写分镜脚本 (write_script_based_on_story)

  Screenwriter.write_script_based_on_story() 将 story 扩展为 scene 列表（即场景脚本），输出 script.json。

  本例生成了 4 个 SCENE，每个对应故事的一个时间段。注意：原始 demo 的 idea 说"不要超过 3 个 scene"，但实际产生了 4 个，说明 user_requirement
  是软约束。

  步骤 5：逐场景生成视频（核心循环）

  for idx, scene_script in enumerate(scene_scripts):
      Script2VideoPipeline(working_dir=f"scene_{idx}")(
          script=scene_script, ...
      )

  每个 scene 在一个独立的 scene_{idx}/ 子目录中工作。本次缓存中只有 scene_0 完成（scene_1~3 应该还在生成中或失败）。

  步骤 6：拼接所有场景视频

  用 moviepy.concatenate_videoclips 将所有 scene 的 final_video.mp4 拼接为顶层 final_video.mp4。

  ---
  第二层：Script2VideoPipeline — 场景内 6 步处理

  这是实际生成视频帧的核心管线。

  步骤 A：设计故事板 (design_storyboard)

  StoryboardArtist.design_storyboard() 将 scene 脚本分解为 ShotBriefDescription 列表，每个包含 cam_idx、visual_desc、audio_desc，输出
  storyboard.json。

  本例 scene_0 分为 6 个 shots（idx 0~5），分配到 4 个摄像机：

  ┌──────┬─────┬──────────────────────────────────┐
  │ Shot │ Cam │               内容               │
  ├──────┼─────┼──────────────────────────────────┤
  │ 0    │ 0   │ 蝴蝶栖息在绿叶上，腿轻轻晃动     │
  ├──────┼─────┼──────────────────────────────────┤
  │ 1    │ 0   │ 蝴蝶扇动翅膀，绒毛在阳光下发光   │
  ├──────┼─────┼──────────────────────────────────┤
  │ 2    │ 1   │ 镜头拉远，蝴蝶飞起，小猫在草地上 │
  ├──────┼─────┼──────────────────────────────────┤
  │ 3    │ 2   │ 奶奶坐在藤椅上抬头看蝴蝶         │
  ├──────┼─────┼──────────────────────────────────┤
  │ 4    │ 2   │ 奶奶脸部特写，微笑说话           │
  ├──────┼─────┼──────────────────────────────────┤
  │ 5    │ 3   │ 广角全景，蝴蝶高飞，所有角色     │
  └──────┴─────┴──────────────────────────────────┘

  步骤 B：分解视觉描述 (decompose_visual_descriptions)

  将每个 shot 的简要描述扩展为详细的 ShotDescription，输出 shots/{idx}/shot_description.json，包括：

  - variation_type: "small" / "medium" / "large" — 决定需要多少关键帧
    - small → 只需 first_frame
    - medium/large → 需要 first_frame + last_frame
  - ff_desc / lf_desc: 首帧/末帧的详细视觉描述
  - ff_vis_char_idxs / lf_vis_char_idxs: 各帧可见角色索引
  - motion_desc: 运动描述
  - audio_desc: 音频描述

  本例 shot_0 的 variation_type 为 "small"（只有腿微小晃动），所以只有 first_frame。

  步骤 C：构建摄像机树 (construct_camera_tree)

  CameraImageGenerator 分析各摄像机之间的关系，构建父子关系树，输出 camera_tree.json：

  Camera_0 (root, Shot 0-1) — 建立核心主体（蝴蝶）
    └─ Camera_1 (child, Shot 2) — parent_cam_idx=0, missing_info="草地+小猫"
         └─ Camera_2 (child, Shot 3-4) — parent_cam_idx=1, missing_info="奶奶+藤椅"
              └─ Camera_3 (child, Shot 5) — parent_cam_idx=2, missing_info="小猫伸爪+金阳光环境"

  关键设计: 子摄像机通过 parent_shot_idx 引用父摄像机的最后一个 shot 作为过渡视频的起点。missing_info 字段描述子镜头中父镜头没有覆盖的新元素。

  步骤 D：生成帧 — generate_frames_for_single_camera（并行核心）

  这是最复杂的步骤，对每个 Camera 并发执行：

  1. 生成首 shot 的 first_frame:
    - 收集可见角色的肖像作为参考图
    - 如果该 Camera 有 parent，先生成 transition_video（从父 shot 到当前 shot 的过渡视频），再从中提取 new_camera_image 作为构图参考
    - 调用 ReferenceImageSelector 从可用参考图中选择合适的参考图并生成图像提示词
    - 调用 image_generator.generate_single_image() 生成 1600x900 的首帧图
  2. 生成首 shot 的 last_frame（如果需要）：
    - 仅 variation_type 为 medium/large 时需要
  3. 生成其他 shot 的帧：
    - 优先生成 priority_shot_idxs（被其他 Camera 引用的父 shot）
    - 然后再生成普通 shot

  帧生成中用 asyncio.Event 做同步——子 Camera 等待父 Camera 的 first_frame 生成完毕。

  步骤 E：生成视频 — generate_video_for_single_shot

  对每个 ShotDescription 并行执行：
  - 等待 first_frame（必需的 Event）
  - 如果 variation_type 为 medium/large，等待 last_frame Event
  - 调用 video_generator.generate_single_video() 传入关键帧（1-2 张）+ motion_desc + audio_desc 作为 prompt

  步骤 F：拼接本场景视频

  用 moviepy 将所有 shot 的 video.mp4 拼接为 scene 的 final_video.mp4。

  完整数据流图

  idea + user_requirement + style
      │
      ▼
  Screenwriter.develop_story()  ──►  story.txt
      │
      ▼
  CharacterExtractor  ──►  characters.json (角色列表)
      │
      ▼
  CharacterPortraitsGenerator  ──►  character_portraits/{idx_name}/front|side|back.png
      │
      ▼
  Screenwriter.write_script_based_on_story()  ──►  script.json (scene 列表)
      │
      ▼  (对每个 scene)
  StoryboardArtist.design_storyboard()  ──►  storyboard.json (ShotBriefDescription 列表)
      │
      ▼
  StoryboardArtist.decompose_visual_description()  ──►  shots/{idx}/shot_description.json
      │
      ▼
  CameraImageGenerator.construct_camera_tree()  ──►  camera_tree.json (Camera 层次)
      │
      ▼  (并行 per Camera)
  generate_frames_for_single_camera()
      ├── 过渡视频 (如有 parent) + new_camera_image
      ├── ReferenceImageSelector 选择参考图+生成 prompt
      └── image_generator  ──►  shots/{idx}/first_frame.png, last_frame.png
      │
      ▼  (并行 per Shot)
  video_generator  ──►  shots/{idx}/video.mp4
      │
      ▼
  moviepy.concatenate  ──►  scene_{idx}/final_video.mp4
      │
      ▼  (所有 scene 完成)
  moviepy.concatenate  ──►  final_video.mp4 (最终输出)
